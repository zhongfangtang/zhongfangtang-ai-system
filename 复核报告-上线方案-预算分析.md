## 二、技术可行性深度分析

### 2.1 平台API对接可行性

| 平台 | 开放API状态 | 自动发布可行性 | 风险等级 | 建议 |
|------|:----------:|:------------:|:------:|------|
| **百家号** | ✅ 有开放API | 🟢 全自动可行 | 低 | 直接对接，API文档完整 |
| **B站** | ✅ 有开放API | 🟢 全自动可行 | 低 | 创作中心API可调用 |
| **视频号** | ⚠️ 仅限企业 | 🟢 全自动可行（企业号） | 中 | 需企业认证，通过微信开放平台 |
| **抖音** | ⚠️ 限制较多 | 🟡 半自动可行 | **高** | 2024年起严格限制自动化，需人工复核 |
| **小红书** | ⚠️ 内测中 | 🟡 半自动可行 | **高** | 开放平台内测，需申请权限 |
| **快手** | ⚠️ 有条件 | 🟡 半自动可行 | **高** | 需服务商资质 |

> **关键结论**: 方案中"半自动"策略是正确的。抖音/小红书/快手三个高风险平台不建议全自动发布，必须保留人工复核环节。全自动模式仅适用于百家号和B站。

## 三、上线路径方案

### 3.1 上线前必须完成的工作（Checklist）

#### 第一阶段：基础设施（1-2天）

- [ ] **购买服务器**: 阿里云/腾讯云轻量应用服务器 2核4G
- [ ] **域名购买+备案**: `zhongfangtang.com` 或子域名（如已有则跳过）
- [ ] **SSL证书**: 免费 Let's Encrypt / 阿里云免费证书
- [ ] **安装Docker**: `curl -fsSL https://get.docker.com | sh`

#### 第二阶段：代码补全（2-3天）

- [ ] **创建缺失文件**:
  - `server/src/workers/scheduler.js` — Worker调度器
  - `deploy/mongo-init.js` — 数据库初始化脚本
  - `deploy/dashboard-nginx.conf` — Dashboard Nginx配置
- [ ] **配置环境变量**: 创建 `server/.env`，填入所有 API Key
- [ ] **AI API 对接**: 将框架代码中的占位逻辑替换为真实 DeepSeek API 调用
- [ ] **平台API对接**: 
  - 百家号/B站优先（低风险全自动）
  - 抖音/小红书/快手后续（需要申请开发者权限）

#### 第三阶段：部署上线（1天）

```bash
# 1. 上传代码到服务器
scp -r zhongfangtang-ai-system/ root@your-server:/opt/

# 2. 配置环境变量
cd /opt/zhongfangtang-ai-system
cp server/.env.example server/.env
vim server/.env  # 填入真实API Key

# 3. 一键部署
docker compose -f deploy/docker-compose.yml up -d

# 4. 验证
curl http://localhost:3001/health
curl http://localhost:8080/
```

#### 第四阶段：业务配置（1-2天）

- [ ] **6平台账号注册/认证**: 完成企业号认证
- [ ] **企业微信配置**: 创建企微应用，配置回调
- [ ] **小程序升级**: 更新API地址指向新后端
- [ ] **内容知识库初始化**: 上传产品资料、话术模板
- [ ] **测试发布流程**: 先在百家号/B站试运行

#### 第五阶段：试运行+优化（1周）

- [ ] **监控配置**: 日志收集、异常告警
- [ ] **内容质量A/B测试**: 对比AI生成 vs 人工创作
- [ ] **截流话术调优**: 根据实际转化数据优化
- [ ] **BI数据校验**: 确认数据准确性

### 3.2 推荐上线节奏

```
第1周: 基础设施 + 代码补全
第2周: 百家号+B站 全自动上线（低风险验证）
第3周: 视频号全自动 + 抖音半自动上线
第4周: 小红书/快手半自动 + CRM/ERP对接
第5周: 全面运行 + 数据优化
```
