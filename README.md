## 一、项目总览与核心目标

### 1.1 项目定位

构建中芳堂专属美业全域AI智能体系统，实现**六大公域平台全流程智能化运营**，达成AI全自动内容生产、分平台智能发布、同城同行精准截流、公域引流私域、全链路留客升单锁客复购，配套全域数据BI可视化分析、CRM深度客户管理、ERP/收银系统数据全同步。

### 1.2 核心目标矩阵

| 维度 | 目标 | 关键指标 |
|------|------|----------|
| **内容生产** | AI全自动 | 日产30+条差异化内容 |
| **平台覆盖** | 6大平台 | 抖音/小红书/视频号/快手/B站/百家号 |
| **获客成本** | 降低80% | 从人均获客200元→40元以下 |
| **私域转化** | 提升300% | 公域→私域留资率>15%，成交率>8% |
| **运营人力** | 减少90% | 从3人团队→0.3人(仅复核) |
| **数据闭环** | 100%可追溯 | 全链路数据打通，BI实时可视化 |

### 1.3 中芳堂业务特色

- **核心业务**: 中医芳香疗法（精油/纯露/膏方）、九体辨识调理、面部皮肤管理
- **核心产品**: 九体辨识复方精油、28养植物精油、伍申堂八款膏方、芝参护肤系列、纤姿达辅助产品、腕家H1健康手表
- **客户分层**: A类(有钱有闲)→到店+居家 / B类(有钱没闲)→纯居家 / C类(有闲抗拒内调)→到店+外用 / D类(钱闲有限)→低价引流培养
- **闭环体系**: AI辨体→方案推荐→居家内调外养→腕家H1数据追踪→到店内调外养→数据对比→方案优化

#### 3.1.1 平台自动化分级

| 平台 | 发布模式 | 日发布量 | 最佳时段 | 内容类型 | 风控等级 |
|------|---------|---------|---------|---------|---------|
| **百家号** | 🟢 全自动 | 3-5条 | 7:00/12:00/18:00 | 图文+视频 | 低 |
| **B站** | 🟢 全自动 | 1-2条 | 12:00/19:00 | 中长视频+图文 | 低 |
| **视频号** | 🟢 全自动 | 2-3条 | 8:00/12:00/20:00 | 短视频+图文 | 中 |
| **抖音** | 🟡 半自动 | 2-3条 | 7:30/12:30/18:30 | 短视频为主 | 高 |

#### 3.1.2 全自动发布流程

```
定时触发 → 内容生成(AIGC) → 合规自检(违规词+图片审核) → 
风险评级 → [低风险] → 自动发布 → 状态回传 → 数据入库
         → [中高风险] → 推送人工复核 → 确认后发布/驳回修改
```

| 小红书 | 200-500字 | ≤5分钟 | 3:4竖图 | 3-8个 | @官方账号+话题 |
| 视频号 | 50-150字 | ≤30分钟 | 9:16/16:9 | 2-4个 | 挂载小程序链接 |
| 快手 | 50-200字 | 11-57秒 | 9:16竖屏 | 3-5个 | 同城+直播预告 |

## 七、风险控制与合规保障

### 7.1 平台风控规避

| 风险点 | 规避策略 |
|--------|---------|
| 内容同质化 | AI差异化生成+随机扰动+人工抽查 |
| 发布频率过高 | 严格按平台限制+随机间隔+分时段 |
| 互动话术违规 | 软性种草模式+禁用医疗功效词+定期更新话术库 |
| 账号关联风险 | 不同平台用不同设备/IP/操作节奏 |
| API调用限流 | 令牌桶限流+指数退避重试+队列削峰 |

### 7.2 合规用语规范

| ✅ 可用 | ❌ 禁用 |
|---------|---------|
| 调理 / 养护 / 改善 | 治疗 / 治愈 / 根治 / 诊断 |
| 美容美体养生服务 | 医疗服务 / 中医诊所 |
| 舒缓 / 放松 / 滋养 | 疗效 / 见效 / 疗程 |
| 参考 / 建议 / 推荐 | 处方 / 医嘱 / 保证 |
| 使用感受 / 个人体验 | 临床验证 / 科学证明 |

### 7.3 数据安全

- 用户数据加密存储（AES-256）
- API通信全量HTTPS
- 敏感信息脱敏展示（手机号/身份证）
- 定期数据备份（每日增量+每周全量）
- 访问权限分级控制（admin/operator/viewer）
- 符合《个人信息保护法》《数据安全法》要求

## 附录: 快速开始

### A. 环境要求

- Node.js 18+
- MongoDB 7.0+ (或 MongoDB Atlas 免费层)
- Redis 7+ (可选，用于任务队列)
- 微信开发者工具 (小程序开发)

### B. 一键部署

```bash
# 克隆/下载项目
cd zhongfangtang-ai-system

# 执行部署脚本
bash deploy/deploy.sh

# 或使用 Docker
docker compose -f deploy/docker-compose.yml up -d
```

### C. 小程序升级

1. 将 `zft-extracted/` 目录导入微信开发者工具
2. 更新 `project.config.json` 中的 AppID
3. 部署云函数（按 DEPLOY.md 说明）
4. 配置 API 地址指向新的后端服务
