## 1. 可直接用的 `.env`（带 🔴 必填标注）

- 文件：`server/.env`（由 `server/.env.example` 生成，已填占位值）
- 用法：把带 🔴 的字段替换为真实值即可运行
- 未配置时系统**自动降级为演示/Mock 模式**启动，不报错
- 必填项速览：

| 配置项 | 说明 | 成本 |
|---|---|---|
| `JWT_SECRET` | 生产随机长字符串 | 免费 |
| `AI_MODEL_API_KEY` | 硅基流动/DeepSeek 密钥 | 免费额度起 |
| 6 组平台密钥（抖音/小红书/视频号/快手/B站/百家号） | 不填则走人工复核队列 | 免费 |
| `WEWORK_*` | 私域对接必填 | 免费 |
| `MINIPROGRAM_APP_ID/SECRET` | 有小程序必填 | 免费 |
| `WEB3_CONTRACT_ADDRESS` / `WEB3_PRIVATE_KEY` | 仅链上「写入」需填 | 测试网免费 |
| `ERP_API_KEY` / `WANJIA_H1_APP_KEY` | 开启深度对接必填 | 免费 |

## 2. 一键启动脚本 `scripts/start.sh`

支持本地与 Docker 两种模式、常用子命令：

```bash
./scripts/start.sh                 # 本地模式（Node 直跑，依赖本地 Mongo/Redis）
./scripts/start.sh --docker        # Docker Compose（自动拉起 mongo/redis/api/dashboard/worker）
./scripts/start.sh stop|restart|status|logs|health
```

能力：检测 Node/Docker → 复制 `.env`（缺失时）→ 安装依赖 → 启动（PM2 或 nohup）→ 轮询 `/health` 健康检查。

## 3. CI/CD 流水线 `.github/workflows/ci.yml`

- 触发：`push` / `PR` 到 `main|master`
- 矩阵：Node 18.x / 20.x
- 步骤：
  1. `npm ci` 安装依赖
  2. **语法检查** `node --check`（零额外依赖）
  3. **单元测试** `node --test`（当前 18 项全部通过）
  4. **Docker 构建校验**（验证 Dockerfile 可构建）
- 部署任务 `deploy` 默认关闭（`vars.ENABLE_DEPLOY=true` 且配置 Secrets 后触发）
- 已将项目初始化为 Git 仓库，并加入 `.gitignore`（忽略 `.env`/`node_modules`/`logs` 等）

## 4. Web3 链上存证引擎

### 后端
- `server/src/services/Web3Service.js`（ethers v6）
  - `computeHash(payload)` → keccak256 内容哈希
  - `notarize({type, payload})` → 写入链上（未配置私钥/合约时**降级为离线哈希**，不报错）
  - `verify(hash)` → 查询哈希是否已上链
  - 支持 `sepolia`(免费测试网) / `mainnet` / `polygon` / `bsc` 一键切换
- `server/src/api/web3Routes.js`（已注册到 `/api/v1/web3`）
  - `GET  /web3/info`       公开：链状态
  - `GET  /web3/verify?hash=` 公开：验证（品牌页调用）
  - `POST /web3/notarize`   鉴权：上链存证
- 配置：`config/default.js` 新增 `web3` 段；`.env` / `.env.example` 新增 `WEB3_*`

### 合约与部署
- `contracts/Notary.sol`：仅存内容哈希（零明文上链，合规且低成本）
- `contracts/deploy.js`：编译后用 `node contracts/deploy.js` 部署，输出地址回填 `WEB3_CONTRACT_ADDRESS`
- 成本：Sepolia 测试网 **免费**；以太坊主网部署一次性约 **¥300-800**，单条存证 Gas 约 ¥20-60

### 前端（品牌体验馆）
- `brand/index.html` 区块链区新增**链上验证输入框 + 按钮**，调用 `/api/v1/web3/verify`
- 后端不可达时优雅降级为「演示模式」提示

## 5. CRM/ERP 深度对接 `IntegrationService`

- `server/src/services/IntegrationService.js`（集成中枢）
  - **企业微信 SCRM**：客户推送为外部联系人、欢迎触达
  - **美业 ERP**（有赞美业/美团收银/客如云/通用 OpenAPI）：客户与订单**双向同步**
  - **腕家 H1 健康手表**：拉取睡眠/心率/血氧等数据进客户档案（`healthMetrics` 字段）
  - **Webhook 接收**：ERP / 小程序订单实时回写 CRM
  - 所有外部系统「未配置即降级」，绝不因缺密钥崩溃
- `server/src/api/integrationRoutes.js`（已注册到 `/api/v1/integrations`）
  - `GET  /integrations`              集成清单
  - `POST /integrations/:name/test`   连通性测试
  - `POST /integrations/sync/erp`     从 ERP 同步
  - `POST /integrations/wanji-h1/:id` 拉取腕家H1数据
  - `POST /integrations/wework/:id`   推送客户到企微
  - `POST /integrations/webhook/:source` 公开 Webhook 接收
- 定时任务（`engines/scheduler.js`）：ERP 每日 03:00 同步；腕家H1 每日 06:00 拉取活跃客户
- 配置：`.env` / `.env.example` 新增 `ERP_*` / `WANJIA_H1_*`

## 6. 无障碍适配（a11y）

共享样式 `assets/a11y.css`，三端统一：

- `index.html`：引入 a11y.css；`<header role="banner">` / `<main id="main" role="main">` / `<footer role="contentinfo">`；跳转链接「跳到主要内容」
- `brand/index.html`：内联 a11y 样式；跳转链接；`hero-canvas` 标记 `aria-hidden`；导航 `aria-label`；尊重 `prefers-reduced-motion`
- `dashboard/index.html`：引用 `../assets/a11y.css`；跳转链接；`<nav>`/`<main>`/`<footer>` 语义标签与 `aria-label`
- 通用规则：键盘焦点可见（`outline`）、减少动效偏好、`.sr-only` 读屏专用、最小字号与行高保证可读性
